## 续学习路径建议（4 周求职冲刺）

结合您的基础，为您规划了 4 周的求职冲刺路径，每个阶段都配套了可放到 GitHub 的实战项目，以下是按思维树模式拆解的详细待办（每一步为一个待办，done默认未完成，可随学习进度修改）：

### 第一周：框架入门（P0 核心）- 核心目标：掌握LangChain/LangGraph，重构ReAct Agent，完成代码助手项目

- Day1：环境准备 - 安装Python（3.10+），配置虚拟环境（conda/pipenv）
- Day1：安装核心依赖 - 安装LangChain、LangGraph、OpenAI（或兼容API）、python-dotenv依赖包
- Day1：验证环境 - 运行LangChain基础示例（简单的LLM调用），确保依赖安装成功
- Day2：LangGraph基础学习 - 阅读LangGraph官方文档，理解核心概念（节点Node、边Edge、状态State、条件分支）
- Day2：LangGraph基础实践 - 编写简单的单节点、多节点流程，实现“提问-回答”基础链路
- Day3：LangGraph状态管理学习 - 理解StateGraph的工作机制，掌握状态的定义、传递和更新方法
- Day3：状态管理实践 - 编写带状态的简单流程（如“提问-记录问题-回答-记录答案”）
- Day4：回顾手写ReAct Agent - 梳理自己之前手写的ReAct Agent代码，明确核心逻辑（Thought/Action/Observation循环、正则解析）
- Day4：LangGraph重构ReAct Agent（第一步）- 定义ReAct的状态结构（包含thought、action、observation、history等字段）
- Day5：LangGraph重构ReAct Agent（第二步）- 编写各个节点函数（思考节点、行动节点、观察节点）
- Day5：LangGraph重构ReAct Agent（第三步）- 定义节点间的边和条件分支（判断是否需要继续行动或直接输出答案）
- Day6：LangGraph重构ReAct Agent（第四步）- 测试重构后的Agent，修复逻辑漏洞（如循环异常、解析错误）
- Day6：LangChain Tools学习 - 阅读LangChain Tools文档，掌握Tool的定义、注册和调用方法
- Day7：LangChain Agent类型学习 - 了解AgentType（ZERO\_SHOT\_REACT\_DESCRIPTION、STRUCTURED\_CHAT\_ZERO\_SHOT\_REACT\_DESCRIPTION等）的区别和适用场景
- Day7：求职项目启动 - 搭建“LangGraph 代码助手Agent”项目结构（创建项目文件夹、初始化README、配置requirements.txt）
- Day7：项目基础搭建 - 实现项目核心入口，完成LLM连接配置（对接OpenAI兼容接口）

### 第二周：RAG 进阶 - 核心目标：掌握高级RAG技术，完成个人知识库Agent项目

- Day1：Hybrid RAG基础学习 - 理解混合检索（BM25+向量检索）的核心原理，明确解决的问题（稀疏向量检索缺陷）
- Day1：依赖安装 - 安装BM25相关依赖（rank\_bm25）、向量检索依赖（chromadb）
- Day2：BM25实践 - 编写简单的BM25检索示例，测试关键词检索效果
- Day2：向量检索回顾 - 复习Embedding与向量相似度计算，用chromadb实现基础向量检索
- Day3：Hybrid RAG实践（第一步）- 整合BM25和向量检索，实现混合检索逻辑（权重分配、结果融合）
- Day3：Hybrid RAG实践（第二步）- 测试混合检索效果，对比纯向量检索、纯BM25检索的准确率
- Day4：进阶分块策略学习 - 学习语义分块、递归分块、基于LLM的分块的原理和适用场景
- Day4：分块实践 - 编写不同分块策略的代码，对比不同分块方式对长文档（1万字以上）的检索效果
- Day5：重排序技术学习 - 了解CrossEncoder、ColBERT模型的核心原理，掌握重排序的作用（提升检索精准度）
- Day5：重排序实践 - 安装CrossEncoder依赖，实现检索结果的重排序，测试效果
- Day6：RAG评估体系学习 - 了解RAG评估的核心指标（召回率、准确率、MRR），掌握评估方法
- Day6：RAG评估实践 - 编写简单的评估脚本，对自己实现的Hybrid RAG进行量化评估
- Day7：求职项目启动 - 搭建“个人知识库Agent”项目结构，配置项目依赖（chromadb、rank\_bm25、CrossEncoder等）
- Day7：项目基础功能实现 - 完成文档上传接口（支持PDF、Markdown格式），实现文档读取功能

### 第三周：多 Agent 与工具生态 - 核心目标：掌握AutoGen，对接MCP工具，完成三Agent系统项目

- Day1：AutoGen基础学习 - 阅读AutoGen官方文档，理解多Agent协作的核心概念（Agent定义、对话流程、角色分配）
- Day1：依赖安装 - 安装AutoGen、pyautogen依赖包，配置LLM接口
- Day2：AutoGen基础实践 - 编写简单的两Agent协作示例（如“提问Agent+回答Agent”），测试对话流程
- Day3：AutoGen角色定义学习 - 掌握不同角色Agent的配置方法（system prompt、能力限制、协作规则）
- Day3：多Agent协作实践 - 实现三Agent简单协作（如“ Planner+Executor+Reviewer”），完成简单任务（如生成简单代码）
- Day4：MCP生态回顾 - 复习自己开发的MCP Server，梳理stdio/sse传输原理
- Day4：MCP社区工具调研 - 查找3-5个社区成熟的MCP Server（如工具市场中的常用工具），了解其接口和功能
- Day5：MCP生态对接（第一步）- 编写MCP客户端代码，实现与第一个社区MCP Server的连接
- Day5：MCP生态对接（第二步）- 测试工具调用功能，确保能正常获取工具返回结果
- Day6：MCP生态对接（第三步）- 依次对接剩余2-4个社区MCP工具，封装统一的工具调用接口
- Day6：测试工具调用稳定性 - 编写测试用例，确保不同工具调用无异常，处理调用失败的异常情况
- Day7：求职项目启动 - 搭建“需求-代码-审查 三Agent系统”项目结构，定义三个Agent的角色和协作规则
- Day7：项目基础功能实现 - 完成三个Agent的初始化配置，实现基础的对话协作流程

### 第四周：工程化部署 - 核心目标：完成项目封装、部署，整理GitHub仓库，更新简历

- Day1：Agent服务化学习 - 学习FastAPI基础，掌握API接口的定义、请求参数和响应格式
- Day1：FastAPI依赖安装 - 安装FastAPI、uvicorn依赖包
- Day2：第一个项目封装（LangGraph代码助手Agent）- 用FastAPI封装Agent核心功能，定义调用接口（如/code\_generate、/code\_fix）
- Day2：测试API接口 - 用Postman或curl测试接口，确保能正常返回结果，处理异常请求
- Day3：第二个项目封装（个人知识库Agent）- 用FastAPI封装文档上传、问答接口，确保接口稳定
- Day3：第三个项目封装（三Agent系统）- 用FastAPI封装协作接口，支持提交需求、获取结果
- Day4：Docker基础学习 - 了解Docker的核心概念（镜像、容器），安装Docker环境
- Day4：编写Dockerfile（第一个项目）- 配置项目依赖、运行命令，构建Docker镜像
- Day5：编写Dockerfile（第二、三个项目）- 分别为两个项目编写Dockerfile，构建镜像
- Day5：测试Docker容器 - 运行三个项目的Docker容器，测试接口是否正常访问
- Day6：编写部署文档 - 为每个项目编写详细的部署步骤（环境准备、Docker构建、容器运行、接口测试）
- Day6：整理GitHub仓库 - 创建GitHub仓库，上传三个项目的代码、Dockerfile、部署文档
- Day7：完善GitHub README - 为每个项目编写README，说明项目功能、技术栈、使用方法、演示效果
- Day7：更新简历 - 在简历中添加三个项目的链接和核心成果，突出LangChain、LangGraph、RAG、多Agent等核心技能
- Day7：复盘总结 - 梳理4周学习内容，查漏补缺，整理学习笔记和项目问题，为面试做准备
